## Commands

```bash
dotnet build                                       # multi-targets net8.0;net9.0;net10.0 for libs, net10.0 for tests
dotnet test                                        # all tests
dotnet test tests/CShells.Tests/                   # unit + integration only
dotnet test tests/CShells.Tests.EndToEnd/          # E2E only
dotnet test --filter "FullyQualifiedName~Foo"      # single test / class
cd samples/CShells.Workbench && dotnet run         # runnable sample at https://localhost:5001/{shell-path}
```

Central package management is enforced via `Directory.Packages.props` — never put `Version=` on a `<PackageReference>`. `Directory.Build.props` and `src/Directory.Build.props` set common project options (multi-target, nullable, implicit usings).

## Spec-kit workflow

Feature work follows the `.specify` flow: `specs/NNN-name/spec.md` → `plan.md` → `tasks.md`. When the user references a feature by number (e.g. "007"), check `specs/007-*/`. Tasks are tracked in that folder's `tasks.md`, not in code.
